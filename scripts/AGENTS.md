# scripts/AGENTS.md

## 脚本目录规则

- 现有脚本是 FLOWR.ROOT 训练, 微调, 生成和亲和力预测的基线入口. 不因修复项目而随意改写其默认行为.
- 根目录 `scripts/` 只放跨实验复用的稳定脚本. 单次实验专用脚本放入 `experiments/<experiment_id>/`.
- 当第一份稳定修复脚本出现时再创建 `scripts/repair/`, 不预先维护空目录.
- 脚本应清楚声明输入, 输出和会产生的文件, 不把正式实验结果写入 `tmp/`.
- 长时间运行或正式实验前, 先用小数据, 短步数或轻量 CPU/GPU 路径验证命令能够正确执行.
