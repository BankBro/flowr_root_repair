# outputs/AGENTS.md

## 输出目录规则

- 实验输出写入 `outputs/<experiment_id>/`, 其中目录名必须与对应的 `experiments/<experiment_id>/` 完全一致.
- 不创建没有 plan 和 experiment 对应关系的孤立输出目录.
- 每个输出目录必须有简短的 `README.md`, 说明来源实验, 内容, 关键结果和保留策略.
- 输出内部结构按实验需要灵活组织, 不强制固定 metadata, schema 或子目录模板.
- 每个输出根目录按实际产物维护局部 `.gitignore`. 默认提交小型总结, 表格和必要的复现信息, 忽略 checkpoint, 大规模逐样本结果, 日志, 缓存和临时文件.
- 不在输出目录维护可跨实验复用的代码. 可复用实现应移入 `flowr/` 或 `scripts/`, 单次实验代码保留在对应实验目录.
- 报告中的结论必须能追溯到本目录中的结果或实验记录, 不只保留脱离上下文的最佳数字.
