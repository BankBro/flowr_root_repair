# flowr/AGENTS.md

## 代码修改规则

- `flowr/` 是 FLOWR.ROOT 主代码目录. 默认训练, 生成, 预测和评估行为必须保持兼容.
- 修复功能采用显式入口或显式参数启用. 未启用修复时, 不得改变原有数据流, loss, 采样流程或输出.
- 新增修复模块优先放入 `flowr/repair/`. 需要接入现有模型, 数据或训练器时, 只增加必要的最小接口.
- 不把首版尚未确定的研究演进模块提前写入主流程. 实现范围以 `docs/STATUS.md` 指向的当前 plan 为准.
- 修改共享代码后, 至少验证受影响的新路径和原有路径. 正式训练前先完成可快速运行的 smoke test.
- 代码中应明确区分 FLOWR.ROOT 原有字段与本课题新增字段, 避免把新增设计描述为官方原生接口.
